CloudTrail Viewer

Desktop application that allows you to analyse and report on API events happening within your AWS Account.

Local Files

Load CloudTrail Event files from your local file system.

S3 Files

Load CloudTrail Event files from an S3 bucket.

Table View

The Simple Table View shows all the Events that have been loaded into the system order by time in ascending order.

It provides a Side Bar that shows the raw Event JSON of any selected event, as well as a tree view of the Event.

The Side Bar also provides a customisable chart where the top X loaded events can be viewed in either a Pie or Bar chart.

Find out more about the Table View

Service Overview

The Service Overview feature shows a breakdown of all events that have been loaded into the application grouped by AWS Service.

Clicking any of the services exposes a Table view where all the features described above are available.

Find out more about Service Overview

User Activity

The User Overview feature breaks the loaded Events down into actions performed by IAM Users.

Clicking any of the Users exposes a Table view where all the features described above are available.

Find out more about User Overview

Resources Events

The Resources Feature allows you to see what Resources have been created or destroyed with the loaded events, these might be new EC2 instances or RDS instances or EC2 being stopped or restarted.

Some API calls have already been provided to get you started, you can customise this feature to only show the events you are interested in.

Security Events

The Security Feature show what API calls have been made in the loaded events that could be considered a security risk, these might be events such as a Security Group being opened or a new Policy Document applied to an IAM object.

Some API calls have already been provided to get you started, you can customise this feature to only show the events you are interested in.

Error Events

The Error Feature shows all the API calls that have generated an Error when they were call grouped together by Error Code.

Help

Each feature provides it's own context aware help, by clicking the ? at the top right of the screen a help window will appear explaining what the current feature does.

Configuration

Using the Preferences menu option there are a couple of configuration options you can make

  • Define AWS accounts providing an IAM key and secret to be able to access a S3 bucket
  • Provide an Alias for your AWS Account ID, useful if you log multiple account into the same S3 bucket
  • Configure what API calls should be displayed on the Security screen
  • Configure what API calls should be displayed on the Resources screen
